DATABASE SCHEMA:

users:
  • id (UUID) - Primary key
  • employee_number (string) - Unique login ID
  • password_hash (string) - bcryptjs hashed
  • is_password_set (boolean) - Password set flag
  • timestamps

jobs:
  • id (UUID) - Primary key
  • title, description, requirements (JSON)
  • application_deadline (DateTime)
  • created_by (FK: users.id)
  • is_active (boolean)
  • timestamps

applications:
  • id (UUID) - Primary key
  • job_id (FK: jobs.id)
  • full_name, email, phone
  • resume_filename
  • extracted_text (Google Vision result)
  • match_score (0-100)
  • evaluation_reasoning (AI explanation)
  • status (PENDING/ACCEPTED/REJECTED)
  • email_sent, email_sent_at
  • timestamps

sessions:
  • id (UUID) - Primary key
  • user_id (FK: users.id)
  • token (JWT)
  • expires_at (DateTime)
  • created_at

extracted_texts:
  • id (UUID) - Primary key
  • application_id (FK: applications.id)
  • original_filename
  • extracted_text
  • extraction_method
  • created_at

RECRUITMENT ALGORITHM:

Input: Resume text + Job requirements
Output: Match score, Decision, Strengths, Gaps, Reasoning

Process:
  1. Extract keywords from job requirements
  2. Search resume text for matches
  3. Calculate match percentage
  4. Apply bonuses:
     • Years of experience: +5
     • Education level: +5
     • Certifications: +3
  5. Apply decision logic:
     • Score ≥70% + min reqs met = ACCEPT
     • Score ≥50% + some reqs met = ACCEPT
     • Otherwise = REJECT
  6. Generate detailed reasoning

Example:
  Job: "Python Developer, 2+ years, REST API"
  Resume: "Python expert, 5 years, REST API developer"
  Score: 75% → ACCEPT
  Reason: "Exceeds qualifications with 5 years experience..."
